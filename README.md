# Redis

## 描述

Redis 是一个开源的、基于内存的数据结构存储系统，可用作数据库、缓存和消息中间件。本基线基于 Redis 构建，提供了企业级的高性能缓存和数据库解决方案，支持多种数据结构、持久化、集群模式、哨兵模式等特性。Redis 以其高性能、丰富的数据类型和原子操作而闻名，广泛应用于缓存、会话存储、实时分析、消息队列等场景。

## 功能

### 核心功能
- **多种数据结构**: 支持字符串、哈希、列表、集合、有序集合、位图、HyperLogLog、地理空间等
- **高性能**: 基于内存操作，支持高并发读写
- **持久化**: 支持 RDB 快照和 AOF 日志两种持久化方式
- **原子操作**: 支持事务、Lua 脚本、原子计数器等
- **过期机制**: 支持 TTL 和过期策略
- **发布订阅**: 支持消息发布订阅模式

### 企业级特性
- **集群模式**: 支持 Redis Cluster 分布式集群
- **哨兵模式**: 支持 Redis Sentinel 高可用方案
- **代理模式**: 支持 Predixy 代理负载均衡
- **监控告警**: 集成 Prometheus 监控和告警
- **日志管理**: 支持结构化日志输出和日志轮转
- **外部访问**: 支持 NodePort 和 Traefik 外部暴露

### 运维功能
- **资源管理**: 支持 CPU 和内存资源限制
- **节点亲和性**: 支持 Pod 反亲和性和节点亲和性配置
- **容忍度配置**: 支持污点容忍度设置
- **健康检查**: 内置健康检查和就绪探针
- **指标导出**: 提供 Prometheus 格式的指标
- **优雅停机**: 支持优雅停机机制

### 高级功能
- **数据分片**: 支持集群模式下的数据分片
- **故障转移**: 支持哨兵模式下的自动故障转移
- **读写分离**: 支持主从复制和读写分离
- **连接池**: 支持连接池管理
- **慢查询**: 支持慢查询日志和分析
- **内存优化**: 支持内存优化和淘汰策略

## 架构

### 部署模式

#### 标准版 (operator-standard)
- **副本数**: 1
- **适用场景**: 开发测试环境、快速部署
- **特点**: 资源占用少，部署简单

#### 高可用版 (operator-highly-available)
- **副本数**: 3
- **适用场景**: 生产环境
- **特点**: 高可用性，故障自动转移

#### 集群模式 (cluster)
- **副本数**: 可配置（建议6个或以上）
- **适用场景**: 大规模数据存储、高并发访问
- **特点**: 数据分片、高可用、高性能

#### 哨兵模式 (sentinel)
- **Redis 副本数**: 可配置（建议2个或以上）
- **Sentinel 副本数**: 可配置（建议3个或以上）
- **适用场景**: 高可用主从复制
- **特点**: 自动故障转移、读写分离

#### 集群代理模式 (cluster-proxy)
- **集群节点**: 可配置
- **代理节点**: 可配置
- **适用场景**: 大规模集群、负载均衡
- **特点**: 统一入口、负载分发

### 资源需求

- **CPU 限制**: 200m

- **CPU 请求**: 100m
- **内存请求**: 256Mi

#### Redis 节点资源（默认）
- **CPU 限制**: 2 Core
- **内存限制**: 4Gi
- **CPU 请求**: 2 Core
- **内存请求**: 4Gi

- **CPU 限制**: 1 Core
- **内存限制**: 4Gi
- **CPU 请求**: 1 Core
- **内存请求**: 4Gi

#### 存储需求
- **数据卷**: 可配置大小
- **访问模式**: ReadWriteOnce

## 使用建议

### 环境选择

#### 开发测试环境
- 使用 **标准版** 基线
- 单节点部署即可
- 减少资源配置
- 适合功能验证和开发测试

#### 生产环境
- **小规模应用**: 使用 **哨兵模式**
- **大规模应用**: 使用 **集群模式**
- **高并发场景**: 使用 **集群代理模式**
- 至少 3 节点部署
- 配置 Pod 反亲和性确保节点分散
- 启用监控告警

### 配置建议

#### 资源规划
```yaml
# 生产环境推荐配置
resources:
  redis:
    limits:
      cpu: "4"
      memory: "8Gi"
    requests:
      cpu: "2"
      memory: "4Gi"
    replicas: 6  # 集群模式建议6个节点
    volume:
      size: 100  # GB
      storageClass: "fast-ssd"
```

#### 存储配置
- 使用 SSD 存储提升性能
- 配置适当的存储类
- 考虑数据增长预留空间
- 建议每个节点至少 100GB 存储

#### 网络配置
- 配置服务发现
- 设置适当的网络策略
- 考虑跨可用区部署
- 配置负载均衡

### 监控告警

#### 关键指标
- **节点状态**: `redis_up`
- **连接数**: `redis_connected_clients`
- **内存使用**: `redis_memory_used_bytes`
- **命令统计**: `redis_commands_processed_total`
- **慢查询**: `redis_slowlog_length`
- **主从复制**: `redis_connected_slaves`

#### 告警规则
- Redis 实例不健康告警 (Warning)
- Redis 故障转移告警 (Warning)
- Redis 连接数过高告警 (Warning)
- Redis 阻塞连接过多告警 (Warning)
- Redis Exporter 宕机告警 (Warning)
- Redis 代理请求响应不匹配告警 (Warning)

### 性能优化

#### 内存优化
- 合理设置 `maxmemory` 和淘汰策略
- 使用 `OBJECT ENCODING` 优化数据结构
- 避免大键和热键
- 定期清理过期数据

#### 网络优化
- 使用连接池
- 批量操作减少网络往返
- 合理设置超时时间
- 使用 Pipeline 减少延迟

#### 持久化优化
- 根据业务需求选择 RDB 或 AOF
- 合理设置 `save` 配置
- 使用 `BGSAVE` 避免阻塞
- 定期备份数据

### 安全配置

#### 访问控制
- 设置强密码
- 配置 ACL 权限
- 限制网络访问
- 定期更换密码

#### 网络安全

- 使用 TLS 加密
- 监控异常访问

5. **备份策略**: 制定数据备份策略

8. **安全配置**: 生产环境必须配置认证
